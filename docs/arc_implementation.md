# ARC (Automatic Reference Counting) Memory Management

В рамках развития компилятора языка Rest мы полностью отказались от старого механизма Borrow Checker-а в пользу неявного и автоматического управления памятью (ARC — Automatic Reference Counting). Это приближает нас к философии разработки под веб (подобно Java Spring Boot), делая работу с памятью безопасной, но при этом прозрачной для пользователя.

## Архитектура ARC

Наш ARC основан на скрытом заголовке (header) счетчика ссылок, размещаемом непосредственно перед данными (payload) в куче.

- **Header Size**: 4 байта (`i32`).
- **Счетчик**: Хранит текущее количество владельцев указателя.
- **Payload**: Сами данные (содержимое структуры, элементы массива, UTF-8 символы строки).
- **Указатели**: Функции и переменные работают исключительно с указателем на *payload*. Указатель на начало блока (нужный для C `free`) вычисляется смещением на 4 байта назад.

## Встроенные Runtime-функции

Компилятор автоматически генерирует в LLVM IR следующие C-совместимые runtime-функции, которые линкуются прямо в конечный бинарник:

1. `*mut u8 __rest_alloc(i32 size)`:
   - Вызывает системный `malloc(size + 4)`.
   - Инициализирует счетчик ссылок (первые 4 байта) значением `1`.
   - Возвращает указатель на 5-й байт (payload).

2. `*mut u8 __rest_retain(*mut u8 ptr)`:
   - Проверяет указатель на `null`.
   - Вычисляет `real_ptr = ptr - 4`.
   - Инкрементирует значение счетчика (`*real_ptr += 1`).
   - Возвращает исходный `ptr` для удобства встраивания в выражения (chaining).

3. `void __rest_release(*mut u8 ptr)`:
   - Проверяет указатель на `null`.
   - Вычисляет `real_ptr = ptr - 4`.
   - Декрементирует счетчик (`*real_ptr -= 1`).
   - Если счетчик падает до `0`, вызывается системный `free(real_ptr)`, тем самым полностью освобождая память.

## Интеграция в Codegen

Автоматическое управление памятью интегрировано напрямую в этап генерации LLVM IR (`src/codegen/compile.rs`), что избавляет AST от ручного трекинга времен жизни (lifetimes). 

### Присваивания (Assignments)
Когда мы переназначаем переменную или поле структуры `A = B;`, происходит следующее:
1. Вызов `__rest_release(A)` для старого значения (чтобы предотвратить утечки).
2. Вызов `__rest_retain(B)` для нового значения.
3. Сохранение `B` в `A`.

### Выход из Scope (Освобождение)
Для каждой переменной, которая выходит из области видимости `{}`, компилятор автоматически добавляет вызов `__rest_release(ptr)`.
Если структура (object) выходит из области видимости или переназначается, компилятор рекурсивно освобождает все ее reference-counted поля (подобно `Drop` в Rust, но генерируется динамически на основе типов полей `HirType`).
