# Архитектура Проекта Rest

Проект **Rest** — это строго типизированный компилируемый язык системного уровня. 
Его главная цель — предоставить разработчикам скорость C/C++ (за счет компиляции через LLVM) при автоматическом управлении памятью (ARC) и прямом доступе к системе через FFI и сырые указатели.

## Структура проекта (Cargo Workspace)

Проект состоит из единственного крейта:

1. **`compiler/` (Крейт `restc`)**
   Содержит исходный код самого компилятора языка Rest.
   - Написан на Rust.
   - Отвечает за полный цикл компиляции исходного `.rest` кода: от лексического анализа до генерации машинного кода через LLVM (`inkwell`).
   - Все runtime-функции (ARC, аллокация, операции со строками) генерируются самим компилятором прямо в LLVM IR — отдельного рантайма на Rust больше нет. Компилятор самодостаточен и опирается только на `libc`.

## Жизненный цикл компиляции

Когда вы запускаете компилятор (например, `cargo run -- web_magic.rest`), происходит следующий конвейер:

1. **Лексический анализ (Lexer):**
   Исходный текст разбивается на токены (ключевые слова, идентификаторы, операторы).

2. **Синтаксический анализ (Parser):**
   Токены собираются в Абстрактное Синтаксическое Дерево (AST). Здесь распознаются функции, структуры, внешние функции и выражения.

3. **Семантический анализ (TypeChecker):**
   Проверка типов (строгая типизация), проверка корректности L-Value для указателей (`&x`), а также проверка возможности приведения типов (кастов).

4. **Lowering (Понижение в HIR):**
   AST трансформируется в High-Level Intermediate Representation (HIR). На этом этапе раскрываются "синтаксические сахара", например `array.length()` или неявные касты.

5. **Генерация LLVM IR (Codegen):**
   Компилятор переводит HIR в LLVM IR. 
   - Здесь автоматически вставляются вызовы функций управления памятью (`__rest_retain`, `__rest_release`).

6. **Компиляция и Линковка:**
   LLVM оптимизирует IR (passes) и генерирует объектный файл (`output.o`). После этого вызывается системный `cc` (C compiler/linker), который линкует объектный файл с `libc`, производя готовый исполняемый бинарник.

## Взаимодействие с ОС (Системные вызовы)

Язык может общаться с операционной системой напрямую через C-совместимый **FFI** (Foreign Function Interface):
- Ключевое слово `extern fn` позволяет декларировать внешние системные вызовы.
- Наличие сырых указателей (`*T`, `&`) позволяет работать с системной памятью и C-структурами (например, `sockaddr`).
