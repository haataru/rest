# Архитектура Проекта

Проект **Rest Compiler** изначально развивался как Workspace из нескольких крейтов, но позже был упрощен (сплющен) в единый монолитный пакет (single-package project) для удобства навигации и сборки.

## Структура Директорий

Весь исходный код компилятора располагается в папке `src/` в корне проекта:

- `src/main.rs` — Точка входа в CLI-приложение (разбор аргументов командной строки с помощью `clap`, обработка ошибок, красивый вывод).
- `src/driver.rs` — Оркестратор компиляции. Он связывает лексер, парсер, семантический анализатор и кодогенерацию LLVM воедино, а также управляет JIT-выполнением.
- `src/lexer/` — Лексический анализатор (токенизатор). Разбивает исходный текст `.rest` на поток токенов.
- `src/parser/` — Синтаксический анализатор (рекурсивный спуск). Преобразует токены в Абстрактное Синтаксическое Дерево (AST).
- `src/sema/` — Семантический анализатор (Type Checker). Проверяет типы, проверяет корректность использования указателей, объявляет переменные.
- `src/ir/` — High-Level Intermediate Representation (HIR). Понижает AST в более простое промежуточное представление для облегчения генерации машинного кода.
- `src/codegen/` — Модуль LLVM кодогенерации (на базе библиотеки `inkwell`). Транслирует HIR в LLVM IR, линкует системные функции, содержит логику ARC (вставка вызовов `retain` / `release`) и JIT-движок.
- `docs/` — Проектная документация.

## Зависимости

Основной стек технологий компилятора:
- **Rust** (компилятор написан на Rust 1.85+).
- **LLVM 18** (через крейт `inkwell` и `llvm-sys`): используется для всей низкоуровневой магии (оптимизации, генерация нативного кода, JIT).
- **Clap**: парсер аргументов командной строки.
- **Anyhow**: система обработки и проброса ошибок.
- **Colored**: окрашивание консольного вывода (CLI-интерфейса) в терминале.

Вся архитектура направлена на максимальную модульность этапов: от текста до машинного кода данные передаются в виде строго типизированных структур (AST -> HIR -> LLVM Module).
