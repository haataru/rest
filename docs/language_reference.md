# Справочник по синтаксису языка Rest

Язык **Rest** спроектирован как системный язык программирования с С-подобным синтаксисом, строгой статической типизацией и автоматическим управлением памятью (ARC). Ниже описаны основные элементы языка.

## 1. Типы данных

### Примитивные типы
* Целочисленные (со знаком): `i8`, `i16`, `i32`, `i64`
* Целочисленные (без знака): `u8`, `u16`, `u32`, `u64`
* Числа с плавающей точкой: `f32`, `f64`
* Логический: `bool` (`true` / `false`)
* Строки: `string` (управляются встроенным ARC)
* Тип-пустышка: `void` (обычно используется для функций без возвращаемого значения)

### Указатели (Raw Pointers)
Rest поддерживает прямую работу с указателями, аналогично C/C++:
* `*T` — указатель на тип `T` (например, `*i32`, `*u8`).

## 2. Переменные и Константы

**Переменные** объявляются с помощью ключевого слова `let`. Тип можно указывать явно, либо он выводится компилятором автоматически (в простых случаях).

```rust
let x: i32 = 10;
let y = 20; // Тип будет определен автоматически
let z: string = "Hello, Rest!";
```

**Константы** объявляются через `const` (их значение вычисляется на этапе компиляции):
```rust
const PI: f64 = 3.14159;
```

## 3. Функции

Для объявления функций используется ключевое слово `def`. Возвращаемый тип указывается после стрелки `->`. Если функция ничего не возвращает, тип можно опустить (или написать `-> void`).

```rust
def add(a: i32, b: i32) -> i32 {
    return a + b;
}

def print_hello() {
    print("Hello!"); // Встроенная функция для вывода
}
```

## 4. Пользовательские типы (Структуры / Модели)

Структуры объявляются с помощью ключевого слова `model`. Они создаются в куче (heap) и их жизнь управляется механизмом ARC.

```rust
model Point {
    x: i32,
    y: i32,
}

def main() {
    // Инициализация модели
    let p = Point { x: 10, y: 20 };
    
    // Доступ к полям
    let sum = p.x + p.y;
}
```

## 5. Управляющие конструкции (Control Flow)

### Условия (If / Else)
```rust
if a > b {
    // ...
} else if a == b {
    // ...
} else {
    // ...
}
```

### Циклы (While / For)
```rust
let i = 0;
while i < 10 {
    i = i + 1;
    if i == 5 {
        continue;
    }
}

// Перебор диапазона:
for x in 0..10 {
    // x примет значения от 0 до 9
}
```

## 6. Взаимодействие с системными библиотеками (FFI)

Вы можете подключать и вызывать внешние С-функции (например, из libc).
```rust
extern fn puts(s: *u8) -> i32;
extern fn malloc(size: u64) -> *u8;
extern fn free(ptr: *u8) -> void;

def main() {
    // Вызов C-функции
    // puts("Hello C!"); // (Осторожно с типами, строки Rest требуют приведения)
}
```

## 7. Низкоуровневые операции (Указатели и Память)

* `&x` — получить адрес переменной (Address Of)
* `*ptr` — разыменование указателя (Dereference)
* `sizeof(T)` — размер типа в байтах
* `as` — явное приведение типов (Cast)

```rust
let val = 42;
let ptr: *i32 = &val; // Взятие адреса
let copy = *ptr;      // Разыменование

let num: i64 = 100 as i64; // Приведение типов
let int_size = sizeof(i32); // Вернет 4
```

## 8. Модули и импорты

Вы можете разбивать код на несколько файлов и импортировать их:
```rust
import "utils.rest";
```

## Как скомпилировать ваш код в Docker

Поскольку для сборки нативного кода (через LLVM) требуются специфические зависимости, проще всего собирать ваш `.rest` код прямо через Docker, используя настроенный образ.

1. Создайте свой файл в корне проекта, например `my_code.rest`:
```rust
model User { id: i32, name: string }

def main() {
    let u = User { id: 1, name: "Admin" };
    // Ваш код...
}
```

2. Соберите Docker-образ (если вы этого еще не сделали):
```bash
docker build -t rest_project .
```

3. Запустите компиляцию вашего файла через компилятор `restc` внутри Docker-контейнера. Эта команда смонтирует текущую папку внутрь докера, чтобы компилятор смог прочитать ваш файл:
```bash
# В PowerShell (Windows):
docker run --rm -v "${PWD}:/app" -w /app rest_project cargo run --release --bin restc -- build my_code.rest

# В командной строке CMD (Windows):
docker run --rm -v "%cd%:/app" -w /app rest_project cargo run --release --bin restc -- build my_code.rest

# В Linux/macOS (Bash):
docker run --rm -v "$(pwd):/app" -w /app rest_project cargo run --release --bin restc -- build my_code.rest

# Если вы хотите сразу запустить код (без создания бинарного файла на диске), используйте подкоманду run
# (пример для CMD):
docker run --rm -v "%cd%:/app" -w /app rest_project cargo run --release --bin restc -- run my_code.rest
```

4. Если в коде нет синтаксических или семантических ошибок, компилятор сгенерирует артефакты (бинарные файлы / объектники / LLVM IR) в зависимости от того, как настроен вывод `restc`. Вы сможете их найти в корне проекта, так как директория была примонтирована.
