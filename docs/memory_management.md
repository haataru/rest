# Управление Памятью (ARC)

В языке **Rest** не используется сборщик мусора (GC), который останавливает выполнение программы для очистки памяти. Вместо этого применяется детерминированный алгоритм **Automatic Reference Counting (ARC)**.

## Как это работает?

Все "сложные" типы данных (на данный момент `string` и структуры `model`) выделяются в куче (heap). Перед самими полезными данными в куче компилятор неявно резервирует небольшой заголовок, в котором хранится счетчик ссылок (reference count) — обычное число.

### Жизненный цикл объекта
1. **Инициализация**: Когда вы создаете структуру `Point { x: 10, y: 20 }` или строку `"Hello"`, выделяется память, а счетчик устанавливается в `1`.
2. **Удержание (`retain`)**: Если вы присваиваете этот объект новой переменной или передаете его в функцию по значению, компилятор неявно вызывает системную функцию `__rest_retain(ptr)`. Она увеличивает счетчик ссылок на 1. При этом сами данные не копируются (работает shallow copy).
3. **Освобождение (`release`)**: Когда переменная выходит из области видимости (scope) — например, функция завершает свою работу, компилятор автоматически вызывает функцию `__rest_release(ptr)`. Она уменьшает счетчик ссылок на 1.
4. **Удаление (Deep Free)**: Если после вызова `__rest_release` счетчик достигает `0`, это значит, что на данные больше никто не ссылается. Память немедленно очищается системной функцией `free()`. Если внутри удаляемой структуры были ссылки на другие ARC-объекты (например, строки), для них тоже каскадно вызывается `release`.

## Безопасность и Сырые указатели

Система ARC работает на уровне типов: `string` и `model` полностью безопасны и никогда не приведут к утечке памяти (за исключением циклических ссылок, которые в текущей версии языка не отслеживаются). 

Однако Rest — это язык системного уровня, поэтому он разрешает обходить ARC с помощью **сырых указателей** (`*T`):
```rust
let val = 42;
let ptr: *i32 = &val;
```
Сырые указатели никак не управляются компилятором. Программист несет полную ответственность за то, чтобы не разыменовать указатель, ссылающийся на освобожденную область памяти.
