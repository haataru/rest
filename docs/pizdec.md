# Известные баги и архитектурные ограничения

## Архитектурное (не реализовано, не баг)

- **Нет модулей** — один файл = одна программа
- **Нет stdlib** — `__ref_strdup` встраивается в каждый модуль, завязка на libc
- **Нет мутабельных ссылок (`&mut`)** — только shared borrow
- **Нет lifetime-аннотаций** — функция не может объявить, что возвращает заимствованное значение
- **Нет дженериков** — `T` как параметр типа отсутствует
- **Нет замыканий** — нет лямбд

## Известные баги

### BUG-A: парсер не принимает `&&`/`||` в условии, когда правый операнд сравнения — переменная

**Симптом:**
```ref
let x: i32 = 5;
let w: i32 = 20;
if x > 0 && x < w {     // parse error: expected `:`, found `(`
    print("ok");
}
```
Литералы справа работают: `if x > 0 && x < 10 { ... }` парсится нормально.  
**Workaround:** разбить на вложенные `if`, либо использовать литеральные правые границы.

### BUG-B: парсер не принимает `for ident in var .. var` с переменными-границами

**Симптом:**
```ref
let lo: i32 = 0;
let hi: i32 = 10;
for i in lo..hi {       // parse error: expected identifier
    print(i);
}
```
Только литеральные границы: `for i in 0..10 { ... }`.  
**Workaround:** хардкодить границы в `for`, переменные использовать только для индексации массива.

### BUG-C: тело `if` внутри `while` ломает парсер на присваивании/вызове/объявлении

**Симптом:**
```ref
let i: i32 = 0;
let grid = i32{1, 0, 1, 0, 1};
while i < 5 {
    if grid[i] == 1 {      // parse error: expected `:`, found `=`
        i = i + 1;
    }
    i = i + 1;
}
```
Внутри `for` работает; вложенные `if` внутри `while` — нет.  
**Workaround:** использовать `for` вместо `while`, либо инвертировать структуру.

### BUG-D: double-free при `s = s + "x"` в цикле

**Симптом:**
```ref
let s = "";
let i: i32 = 0;
while i < 5 {
    s = s + "a";        // free(): double free detected in tcache 2
    i = i + 1;
}
print(s);
```
В прямом коде (вне цикла) `s = s + "x"` работает корректно.  
**Workaround:** использовать `let s = s + "x"` вместо `s = s + "x"`, либо собирать строку через `string[]` массив с индексным присваиванием (см. `examples/game_of_life.rf`).
